Webtrends Segments

Problem

Several products in the Webtrends ecosystem needed a way to specify specific groups of people either for testing or data analysis. The industry term for these groups is segments. Webtrends users required a place to create and manage user segments from which the segments could be accessible to all Webtrends products.

Solution

  1. Using customer feedback, competitive research, and knowledge of the Webtrends development roadmap we created a feature set and list of requirements.
  2. This being the first of many asset management interfaces we designed workflows and wireframes to be object-agnostic.
  3. We created several wireframes featuring varying interactions for building segments. Through user tests and developer elaboration we decided on the one shown here, using a slider menu and drag-and-drop interaction.
  4. The beta was released to customers and we began collecting feedback and usage statistics which drove further design and development.

My Role

As the Lead Designer on Segments I was responsible for all design elaboration, usability tests, wireframes, pixel designs, and CSS.
